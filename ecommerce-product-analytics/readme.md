# E-commerce Product Analytics

## Описание проекта

Проект посвящен анализу пользовательского поведения в e-commerce на основе событийных данных.

В работе рассматривается поведение пользователей на разных этапах взаимодействия с интернет-магазином: просмотр товаров, добавление в корзину и покупка. Основной фокус сделан на продуктовой аналитике, оценке конверсии, сегментации пользователей, анализе выручки, удержании и демонстрации A/B-тестирования.

## Цель проекта

Провести продуктовый анализ e-commerce данных и определить, какие группы пользователей дают наибольшую ценность для бизнеса.

## Основные задачи

- загрузить и подготовить данные;
- изучить структуру пользовательских событий;
- построить воронку пользовательского поведения;
- рассчитать конверсию между этапами;
- сегментировать пользователей по активности;
- оценить вклад сегментов в выручку;
- рассчитать cohort retention;
- провести демонстрационный A/B-анализ;
- сформулировать продуктовые выводы.

## Используемые данные

В проекте используется датасет eCommerce behavior data from multi category store.

Датасет содержит события пользователей интернет-магазина:
- просмотр товара;
- добавление товара в корзину;
- покупка;
- идентификатор пользователя;
- идентификатор товара;
- категория товара;
- цена;
- время события.

Файлы данных не добавлены в репозиторий из-за размера.

## Структура анализа

### 1. Подготовка данных

На этом этапе выполняется загрузка данных, объединение выборок, проверка типов данных и подготовка временных признаков для дальнейшего анализа.

### 2. Анализ событий

Исследуется распределение событий по типам. Это позволяет понять, как пользователи взаимодействуют с платформой и какие действия встречаются чаще всего.

### 3. Воронка поведения

Строится пользовательская воронка:

- view;
- cart;
- purchase.

На основе воронки оценивается переход пользователей между ключевыми этапами.

### 4. Конверсия

Рассчитываются показатели конверсии между этапами пользовательского пути. Это помогает выявить этапы, где теряется основная часть пользователей.

### 5. Сегментация пользователей

Пользователи разделяются на группы по уровню активности. Для каждого сегмента оцениваются поведение, покупки и вклад в выручку.

### 6. Анализ выручки

Оценивается распределение выручки по пользовательским сегментам. Это позволяет определить наиболее ценные группы пользователей.

### 7. Cohort retention

Рассчитывается удержание пользователей по когортам. Анализ показывает, какая доля пользователей возвращается в продукт после первого периода активности.

### 8. Демонстрационный A/B-анализ

Проводится пример статистической проверки различий между двумя группами пользователей. Блок показывает подход к оценке изменений в продукте на основе статистических методов.

## Основные выводы

- Большая часть пользователей ограничивается просмотром товаров.
- До покупки доходит небольшая доля аудитории.
- Наиболее активные пользователи дают больший вклад в выручку.
- Сегментация пользователей помогает выделить группы с разной продуктовой ценностью.
- Cohort retention позволяет оценить повторную активность пользователей.
- A/B-анализ помогает проверять продуктовые гипотезы на основе статистики.

## Ограничения проекта

- Анализ выполнен на ограниченной выборке данных.
- A/B-тест является демонстрационным и не основан на реальном эксперименте.
- Выводы по пользовательскому поведению требуют дополнительной проверки на уровне сессий и отдельных товаров.
- Данные не включены в репозиторий из-за размера.
