# Рекомендательная система фильмов

## Описание проекта

Проект представляет собой учебный веб-сервис для рекомендации фильмов. Пользователь входит в систему, просматривает список фильмов, выставляет оценки и получает персонализированные рекомендации.

Проект реализован как небольшое full-stack приложение. Backend написан на FastAPI, данные хранятся в SQLite, для работы с базой используется SQLAlchemy. Frontend реализован на HTML и JavaScript.

## Цель проекта

Цель проекта, показать базовую реализацию рекомендательной системы и продемонстрировать навыки работы с backend-разработкой, базами данных, API и алгоритмами рекомендаций.

## Основная функциональность

- создание и хранение пользователей
- хранение списка фильмов
- добавление пользовательских оценок
- получение списка всех фильмов
- получение персонализированных рекомендаций для пользователя
- работа через REST API
- простой веб-интерфейс для взаимодействия с системой

## Описание основных файлов

`backend/main.py` содержит FastAPI-приложение и основные API-эндпоинты.

`backend/database.py` отвечает за подключение к SQLite-базе данных.

`backend/models.py` содержит SQLAlchemy-модели таблиц пользователей, фильмов и оценок.

`backend/schemas.py` содержит Pydantic-схемы для проверки входных и выходных данных API.

`backend/crud.py` содержит функции для работы с пользователями, фильмами и оценками.

`backend/recommender.py` содержит алгоритм формирования рекомендаций.

`backend/populate_data.py` используется для заполнения базы тестовыми данными.

`frontend/index.html` и `frontend/script.js` реализуют простой пользовательский интерфейс.

`DBStruct.drawio` содержит схему структуры базы данных.

## Структура базы данных

В проекте используются три основные таблицы.

### Users

Таблица пользователей.

Основные поля:

- user_id
- username
- password

### Movies

Таблица фильмов.

Основные поля:

- movie_id
- title
- genre
- year

### Ratings

Таблица оценок фильмов пользователями.

Основные поля:

- rating_id
- user_id
- movie_id
- rating

Связи:

- один пользователь может поставить много оценок
- один фильм может иметь много оценок
- таблица Ratings связывает пользователей и фильмы

## Алгоритм рекомендаций

В проекте реализован подход user-based collaborative filtering.

Логика работы алгоритма:

1. Из базы данных загружаются оценки пользователей.
2. На основе оценок строится матрица пользователь-фильм.
3. Пропущенные значения в матрице заполняются нулями.
4. Между пользователями рассчитывается cosine similarity.
5. Для выбранного пользователя находятся наиболее похожие пользователи.
6. На основе оценок похожих пользователей рассчитываются средние оценки фильмов.
7. Из рекомендаций исключаются фильмы, которые пользователь уже оценил.
8. Пользователю возвращается список рекомендованных фильмов.

## Установка и запуск

Перейдите в папку проекта:

```bash
cd Recomendation_system_project
```

Установите зависимости:

```bash
pip install fastapi uvicorn sqlalchemy pandas numpy scikit-learn
```

При необходимости заполните базу тестовыми данными:

```bash
python -m backend.populate_data
```

Запустите backend:

```bash
uvicorn backend.main:app --reload
```

После запуска API будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Документация FastAPI будет доступна по адресу:

```text
http://127.0.0.1:8000/docs
```

Для запуска frontend откройте файл:

```text
frontend/index.html
```

## Пример пользовательского сценария

1. Пользователь вводит имя в интерфейсе.
2. Система создает пользователя или находит существующего.
3. Пользователь просматривает список фильмов.
4. Пользователь выставляет оценки фильмам.
5. Система рассчитывает рекомендации.
6. Пользователь получает список рекомендованных фильмов.

## Ограничения проекта

Проект является учебным и демонстрирует базовую механику рекомендательной системы.

Основные ограничения:

- используются тестовые данные
- авторизация реализована в демонстрационном виде
- пароли не хешируются
- модель рекомендаций не оценивается через метрики качества
- рекомендации пересчитываются при запросе
- не реализована отдельная обработка новых пользователей без оценок

## Возможные направления развития

## Итог
